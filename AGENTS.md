# Dotfiles Management System вЂ” Agent Guide

## Project Overview

This repository manages dotfiles, packages, installations, system configurations, and environment setup across multiple platforms. It is designed to work after a fresh OS install, on a new machine, or on a VPS.

## Key Tools

| Tool | Role |
|------|------|
| **chezmoi** | Dotfiles source state, machine classification prompts, lifecycle scripts |
| **mise** | User-space development tools and languages |
| **Ansible** | System-wide configuration, services, desktop environment settings |
| **WinGet DSC** | Windows native declarative package management |

## Architecture Decisions

### Machine Classification
- Uses chezmoi's `promptChoiceOnce` and `promptMultichoiceOnce` for interactive classification
- Categories: `work`, `personal`, `gaming`, `multimedia`, `development` (multi-choice)
- Form factors: `desktop`, `laptop`, `server`
- Data stored in `~/.config/chezmoi/chezmoi.yaml` (auto-persisted by chezmoi)

### Data Flow
1. `chezmoi init` triggers `hooks.read-source-state.pre` в†’ installs prerequisites (git, curl, yq, Bitwarden CLI)
2. `chezmoi apply` runs prompt functions в†’ writes classification data
3. `run_once_after_10-install-managers` в†’ installs mise + Ansible
4. `run_after_10-install-packages` в†’ installs packages from `.chezmoidata/packages.yaml`
5. `run_after_20-provision` в†’ runs `mise install`, generates Ansible inventory, runs Ansible playbook

### Package Priority
1. **mise** вЂ” if tool is in mise registry (defined in `dot_config/mise/config.toml`)
2. **WinGet DSC** вЂ” on Windows native (`dot_config/winget.dsc.yaml`)
3. **Ansible** вЂ” system packages via roles (`ansible/roles/<role>/`)
4. **Chezmoi scripts** вЂ” fallback, declared in `.chezmoidata/packages.yaml`

### Windows Support
- **Native Windows 11**: Uses PowerShell + WinGet DSC + PowerShell scripts. Ansible is not run.
- **WSL**: Runs the full Linux flow. Ansible targets both WSL `localhost` and the Windows native host via SSH.

### Git Submodules в†’ Chezmoi Externals
- Uses `.chezmoiexternal.yaml.tmpl` instead of git submodules
- Allows template conditionals based on machine classification
- Supports private repos via Bitwarden-stored GitHub PAT

## Testing Commands

After making changes, always verify:

```bash
# Chezmoi health checks
chezmoi doctor
chezmoi data
chezmoi execute-template '{{ .category }}'

# Mise health check
mise doctor

# Dry runs
chezmoi apply --dry-run --verbose
chezmoi apply --dry-run --verbose --source-path chezmoi/.chezmoiscripts/run_after_20-provision.sh
```

## Idempotency

All scripts are designed to be idempotent:
- Check if a package is already installed before installing
- Check if a service is already running before starting
- Use `run_once_*` where appropriate
- Regenerate `inventory.yaml` completely on each run (no append logic)

## Notes for AI Agents

- When adding new tools, prefer mise if available in its registry
- For Windows native, prefer WinGet DSC over Chocolatey/Ansible
- Ansible roles should be single-responsibility (one role per package/config)
- Use `ansible.builtin.group_by` in `pre_tasks` for dynamic grouping
- Always use `.yaml` extension, never `.yml`
- Keep the bootstrap minimal вЂ” let chezmoi scripts handle the heavy lifting
